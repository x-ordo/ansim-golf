# Definition of Ready (DoR)

> 작업을 스프린트 백로그에 포함하거나 개발을 시작하기 위해 충족해야 하는 최소 기준입니다.

## 1. User Story Check
- [ ] **Standard Format**: "As a [User], I want [Feature], So that [Benefit]" 형식인가?
- [ ] **INVEST**: Independent, Negotiable, Valuable, Estimable, Small, Testable한가?
- [ ] **Acceptance Criteria**: 'Given-When-Then' 형식의 명확한 수락 기준이 있는가?

## 2. Resources & Dependencies
- [ ] **Design**: UI/UX 목업 또는 프로토타입(Figma)이 준비되었는가?
- [ ] **API**: 필요한 백엔드 API 명세(Swagger/OpenAPI)가 정의되었는가?
- [ ] **Data**: 사용할 데이터 모델(Schema)과 샘플 데이터가 있는가?

## 3. Risk & Compliance
- [ ] **Security**: 보안 위협 요소(Auth, PII 등)가 검토되었는가?
- [ ] **Legal**: 법적 제약 사항(약관, PG 정책 등)이 해결되었는가?
